1.お客さんからの問い合わせ

「インターネットが見れないんですけど」と問い合わせがありました。問題を解決するのにどのような意図でどのような質問をしていきますか？

問題が端末にあるのか回線にあるのかを以下のように割り出し対処します。

  • 同じ場所で他の端末でもつながらない場合は、回線側に問題がある可能性が高い
  • 違う場所で同じ端末ではつながる場合は、回線側に問題がある可能性が高い
  • 同じ場所で他の端末ではつながる場合は、端末側に問題がある可能性が高い
  • 違う場所で同じ端末でもつながらない場合は、端末側に問題がある可能性が高い
  • 他の場所や他の端末が無い場合は判断がつかないため、端末の問題と回線の問題のどちらの可能性も考慮
【回線側に問題がある可能性が高い場合】
『キャリア回線』
電波の受信状況は良いか？
電波の受信状況の良いところまで出ていただく
障害情報は出ていないか？
こちらで障害情報を確認し、障害ある場合は復旧するまで待っていただく
『Wi-Fi・有線』

一度ルーター、ONU・モデムを再起動していただき、再起動しても繋がらない場合は以下を試していただく。

1.ルーターの赤ランプは点滅していないか？

赤ランプが点滅していたらルーターがネットワークにつながっていないため、ルーターまでの回線の不具合が考えられるので以下を試す

ONU・モデムからルーターまでの有線LANが断線していないか？
ONU・モデムからルーターまでの有線LANを替えていただき、それでもつながらない場合は回線業者側の不具合の可能性があるため、回線業者へお問い合わせいただくようご案内
2.ONUやモデムから有線で繋げられる場合は有線で繋ぐ
つながらない場合はONUやモデムの不具合、もしくはONUやモデムまでの線の不具合が考えられるので、回線業者にお問い合わせいただくようご案内。
3.ルーターから有線で繋げられる場合は有線で繋ぐ
ONUやモデムから有線ではつながったが、ルーターから有線はつながらない場合は、ルーターまでのLANが断線しているか、ルーターの不具合が考えられるので、ルーターまでのLANを交換していただき、それでもだめならルーターの買い替えをご案内
ルーターから有線でつながるがWi-Fiでつながらない場合は、ルーターの不具合か、ルーターの設定が正しくない可能性が考えられるので、ルーターの設定を見直すようご案内
4.ルーターの設定等を見直す
ルーターのファームウェアのアップデートや、設定の初期化等を試しても繋がらない場合は、ルーターのWi-Fi機能が死んでいる可能性が考えられるので、ルーターの買い替えをご案内

有線で確認ができない場合は、ファームウェアのアップデートや、設定の初期化等を試してもらい確認する。
端末にWi-Fiのマークが波紋状に出ているのに繋がらない場合は、ルータより前の回線の不具合が考えられるので、通信業者にお問い合わせいただくようご案内
端末にWi-Fiのマークが出ない場合は、ルーターの不具合が考えられるので、ルーターの買い替えをご案内

2.応答が返ってこない

大量にDBに保存されている売上データに対し集計を掛けましたがいくら経っても応答が返ってきません。何をどのように探り解決していきますか？

急に応答が帰ってこなくなったのか、徐々に応答が遅くなってきていたか確認します。

急に応答が帰ってこなくなった場合、データベース側ではなく、インフラ側に何か問題がある可能性があります。

インフラ側で何か問題が起こっていないか、直近で変更をしたか確認します。

徐々に応答が遅くなってきていた場合や、他のSQL文では応答が帰ってくる場合は、データベース側に問題がある可能性が高いと判断します。

以下を確認します。

SQL文は最適化されているか？
SELECT文が多様されていないか、JOIN句を用いることができないかなどを確認する
複合インデックスを利用しているか？
WHERE句などでよく一緒に利用する列は複数インデックスし、実行速度を上げる
複合インデックスの2列目以降をWHERE句条件に指定していないか？
2列目以降をWHERE句条件に指定しているとインデックスが効かないので確認する
複合インデックスの走査範囲を考慮しているか？
走査範囲が狭い方を複合インデックスの1列目に指定する
LIKE句の前方にワイルドカードがついていないか？
ワイルドカードの前までしかインデックスが効かないので、ワイルドカードはなるべく後ろに持ってくる
大量のソートが発生していないか？
極力インデックスを張った列をORDER BYやGROUP BYに使用する
インデックスできない場合は適切なWHERE句を書く
頻繁に発行されるクエリが多発していないか？
クエリが頻繁に発行されるとCPUが常時稼働するためパフォーマンスに悪影響が出るので、SQLの実行回数を減らせないか検討する

プログラム問題

初めに

  • 回答で作ったプログラムはtest.jsもしくは新しく任意にファイルを作成し記述し、このHTMLファイルから呼び出されるようにすること
  • 作ったプログラムの出力は、背景が緑の黒い枠で囲まれたエリアに出力すること(4の問題は除く)
  • Chromeの最新で動作すればOKとする
  • ES6やTypeScriptで記述してもOKとする
  • 別のライブラリなどを組み込んで使用してもOKとする
  • 良いと思った仕様を追加してもOKとする
  • 検索してもOKだが、1の問題は出来ないことが開発者になってはいけないレベルで恥ずかしいのでプログラムの文法以外は自力で考えることを推奨
  • 問題に対する質問は自由

1.FizzBuzz問題

1から100までの数をプリントするプログラムを書け。ただし3の倍数のときは数の代わりに「Fizz」と、5の倍数のときは「Buzz」とプリントし、3と5両方の倍数の場合には「FizzBuzz」とプリントすること。

2.金種計算問題

金額を引数に渡したら、金種を計算し、それぞれの枚数を出力するプログラムを書け。使用する金種は10000円、5000円、1000円、500円、100円、50円、10円、5円、1円の9種とする。

出力例

19769 => 1=4,5=1,10=1,50=1,100=2,500=1,1000=4,5000=1,10000=1
1234 => 1=4,5=0,10=3,50=0,100=2,500=0,1000=1,5000=0,10000=0
9281 => 1=1,5=0,10=3,50=1,100=2,500=0,1000=4,5000=1,10000=0
